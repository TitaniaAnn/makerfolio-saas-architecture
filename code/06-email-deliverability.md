# 06 — Email & deliverability

**Audience:** developers working on makerfolio-saas.
**Scope:** how outbound mail is composed, dispatched across drivers, kept DMARC-aligned via a verified platform sender, and how SES bounces/complaints come back in.

> All outbound mail funnels through `Mailer`, which assembles RFC-822 headers and hands them to one of four drivers (`mail`/`log`/`smtp`/`ses`) chosen by the `MAIL_DRIVER` env. On SaaS the `From` is always a SES-verified identity — `noreply@PLATFORM_DOMAIN`, or a Studio tenant's own verified domain — never the tenant's (usually unverified) `contact_email`, which would fail SPF/DKIM/DMARC at the receiver and get spam-foldered. SES bounce + complaint feedback returns over SNS to [`public/ses-webhook.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/ses-webhook.php), recorded idempotently in `public.email_bounces`.

## The flow

**Composition.** Most user-facing mail renders an [`EmailTemplates`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/EmailTemplates.php) entry (`owner_new_order`, `customer_receipt`, `customer_shipped`, `admin_password_reset` — [EmailTemplates.php:32](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/EmailTemplates.php#L32)) with a `{var}` bag; admins can override subject/body via `settings` rows keyed `email.{key}.{field}` ([EmailTemplates.php:174](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/EmailTemplates.php#L174)). Platform/system notices (signup verification, suspension, dunning, domain lifecycle, transfer) are composed inline in `Mailer` because they fire before/around the tenant's own template store exists ([Mailer.php:171](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L171)).

**Header assembly.** Every `Mailer` helper builds a `From:` + `Content-Type:` (+ optional `Reply-To:`) header string, CRLF-sanitizing each field through `sanitizeHeader()` ([Mailer.php:607](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L607)) — the header-injection guard — then calls the private `dispatch()` ([Mailer.php:507](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L507)).

**Driver selection.** `dispatch()` branches on `MAIL_DRIVER` ([Mailer.php:508](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L508)):
- `log` — appends the rendered envelope+body to `MAIL_LOG_PATH` ([Mailer.php:509](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L509)); lets dev/CI/smokes read the signup-verify URL with no MTA.
- `smtp` — parses From/Reply-To back out of `$headers` and calls `SmtpMailer::send()` ([Mailer.php:519](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L519)).
- `ses` — parses headers, applies the per-tenant identity override, calls `SesMailer::send()` ([Mailer.php:534](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L534)).
- default `mail` — PHP `mail()` ([Mailer.php:579](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L579)).

The `smtp`/`ses` paths recover From/Reply-To via `parseFromHeaders()` ([Mailer.php:588](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L588)) rather than threading new params through 15+ callers — every caller already stamps the headers the `mail()` path needs, so centralising the parse keeps `Mailer`'s public API unchanged.

**From = verified sender.** In `send()` (shop-order path) the `From` address is `noreply@PLATFORM_DOMAIN` whenever `PLATFORM_DOMAIN` is set, else `contact_email` (self-host MTA) ([Mailer.php:469](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L469)). `PLATFORM_DOMAIN` comes straight from env ([config.php:154](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/config/config.php#L154)) — empty string means single-tenant/self-host and reverts to the inherited behavior.

**Reply-To = tenant.** Because `From` is a no-reply, the maker stays reachable via `Reply-To`: `send()` defaults it to `contact_email` on SaaS unless the caller passed one ([Mailer.php:476](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L476)). The owner-new-order mail instead sets `Reply-To` to the *customer* so the maker can reply straight to the buyer ([Mailer.php:39](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L39)).

**SES transport.** `SesMailer::send()` builds the `Source` (display-name-quoted), `Destination`, text `Message`, optional `ReplyToAddresses`, and `ConfigurationSetName` (the hook that delivers bounce/complaint events to SNS) ([SesMailer.php:39](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SesMailer.php#L39)). AWS errors are caught and return `false` so a SES rate-limit/sandbox blip never crashes the caller ([SesMailer.php:74](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SesMailer.php#L74)).

**SMTP transport — alt ports.** Hetzner blocks outbound 25/465, so SMTP submission uses STARTTLS on 587 (`SMTP_SECURITY=tls`) or implicit TLS on 465 (`ssl`) ([SmtpMailer.php:38](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SmtpMailer.php#L38), [config.php:177](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/config/config.php#L177)). Peer verification is on ([SmtpMailer.php:39](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SmtpMailer.php#L39)); `buildMessage()` CRLF-normalises and dot-stuffs the body ([SmtpMailer.php:134](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SmtpMailer.php#L134)).

**Per-tenant sender (Studio).** In the `ses` branch, if a tenant context is active (`TenantResolver::currentTenantId()`) and `SenderIdentity::forSending($tid)` returns a row, its `from_address`/`from_name` *override* the platform `From` ([Mailer.php:561](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L561)). `forSending()` returns the active identity only when `status = VERIFIED` **and** the tenant's plan has `allow_custom_email_sender` ([SenderIdentity.php:62](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SenderIdentity.php#L62)) — the plan gate makes a downgrade silently drop the custom sender without needing per-identity cleanup. Identity DNS/DKIM lifecycle runs through the `PENDING_DNS → VERIFIED → FAILED → DISABLED` state machine ([SenderIdentity.php:226](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SenderIdentity.php#L226)), driven by SES status polls via `applyVerificationStatus()` ([SenderIdentity.php:136](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SenderIdentity.php#L136)).

**Feedback loop.** SES publishes bounce/complaint events to SNS, which POSTs JSON to [`public/ses-webhook.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/ses-webhook.php). The handler validates the SNS signature with the AWS SDK ([ses-webhook.php:69](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/ses-webhook.php#L69)) — load-bearing: without it a forged POST could mark identities FAILED. It maps `mail.source` back to a sender-identity row ([ses-webhook.php:147](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/ses-webhook.php#L147)) and INSERTs into `public.email_bounces` with `ON CONFLICT (sns_message_id) DO NOTHING` for idempotent retries ([ses-webhook.php:220](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/ses-webhook.php#L220)).

## The verified-sender contract

**The rule:** the SMTP envelope `From` must be a sender identity SES has verified — the platform identity `noreply@PLATFORM_DOMAIN`, or (Studio) the tenant's own VERIFIED domain. It must *never* be the tenant's `contact_email`, which is typically an unverified third-party address. SES rejects/spam-folders unverified `From`s, and a `From` outside an aligned, DKIM-signed domain fails DMARC at the receiver.

This was retrofitted across the two surfaces that previously sent from `contact_email`:
- **Password reset** — `sendPasswordReset()` now derives `From` from `PLATFORM_DOMAIN` ([Mailer.php:147](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L147)); the comment ([Mailer.php:140](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L140)) records that the old `?: $to` fallback could put the *recipient's own* address in `From` and fail DMARC. Commit `063149a` (PR #103); the reset *link* host fix is `725a345` (PR #104).
- **Shop order emails** — the shared `send()` was moved to the platform `From` with `contact_email` as `Reply-To` in commit `da056b5` (PR #132) ([Mailer.php:458](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L458)).

> **`Mailer::send()` parameter order — the subtlety.** The private signature is `send(string $to, string $subject, string $body, string $replyTo = '')` ([Mailer.php:458](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L458)). **The 4th argument is `$replyTo`, NOT a `from` address.** Callers cannot override the `From` here — it is always computed inside `send()` as the verified platform sender. The only way to send `From` a tenant's address is the SES per-tenant identity override in `dispatch()`. Passing a tenant's `contact_email` as that 4th arg sets `Reply-To`, which is exactly the contract: maker reachable, sender verified.

## Invariants & gotchas

- **From is always SES-verified** — `noreply@PLATFORM_DOMAIN` or a tenant VERIFIED identity, never `contact_email`. Break it and SES rejects or DMARC fails → spam folder. ([Mailer.php:469](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L469))
- **`send()`'s 4th arg is `$replyTo`** — not `from`. Don't read it as a sender override; there is no per-call `From` override outside the SES identity path. ([Mailer.php:458](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L458))
- **Every header field passes `sanitizeHeader()`** — strips CRLF + control chars to block header injection (a CRLF in a name could inject `Bcc:`). ([Mailer.php:607](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L607); test [MailerTest.php:17](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/tests/MailerTest.php#L17))
- **Self-host (empty `PLATFORM_DOMAIN`) keeps old behavior** — `From` falls back to `contact_email` (their own MTA, no SES). The SaaS branches are all `PLATFORM_DOMAIN !== ''`-guarded. ([config.php:151](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/config/config.php#L154))
- **Tenant identity gated by VERIFIED + plan** — `forSending()` returns null on either failure, so a Studio downgrade or a never-verified domain falls back to the platform identity without manual cleanup. ([SenderIdentity.php:62](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SenderIdentity.php#L62))
- **Identity lookup is fail-soft** — a Postgres hiccup in the `dispatch()` tenant-identity lookup is caught and falls back to the platform identity rather than dropping the mail. ([Mailer.php:554](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L554))
- **SNS signature validation is mandatory** — the webhook 403s unvalidated POSTs; missing AWS SDK classes 500 every POST (safe-by-default). ([ses-webhook.php:54](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/ses-webhook.php#L54), [ses-webhook.php:69](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/ses-webhook.php#L69))
- **Bounce INSERT is idempotent per SNS MessageId** — `ON CONFLICT DO NOTHING`; multi-recipient bounces suffix the index (`{id}#{i}`) so 3 addresses get 3 rows without aborting the tx. ([ses-webhook.php:218](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/ses-webhook.php#L218))
- **`ses` driver never loads the SDK on `mail`/`log`/`smtp`** — `require_once` is inside the branch, so light paths stay SDK-free. ([Mailer.php:535](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Mailer.php#L535))
