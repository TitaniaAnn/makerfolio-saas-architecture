# 04 — Billing: the platform plane (Pro/Studio subscriptions)

**Audience:** developers working on makerfolio-saas.
**Scope:** the platform Stripe account — how a tenant upgrades to a paid plan (Pro/Studio), and how webhook events drive `subscriptions.local_status` + `tenants.plan_id`.

> makerfolio runs two entirely separate Stripe planes. This walkthrough is the **platform** plane: makerfolio's own Stripe account, billing tenants for their subscription tier. The other plane — each tenant's Stripe **Connect** account for their storefront checkout — is [`05-shop-connect-plane.md`](./05-shop-connect-plane.md). Different accounts, secret keys, webhook endpoints, and DB tables. They can both be active in one request without interference ([`PlatformStripe.php:6-10`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformStripe.php)).

## The flow
Each billing controller gates on `Auth::requireLogin()` + `require_role(Role::OWNER)` (see [`02-auth-and-security.md`](./02-auth-and-security.md)) — billing is OWNER-only.

1. **Plans page** — [`upgrade.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/upgrade.php) lists `public.plans WHERE is_active = 1` ([`upgrade.php:20`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/upgrade.php)). Each paid card's CTA is disabled when `COMING_SOON_PAID_PLANS`, `!PlatformStripe::enabled()`, or the plan has no `stripe_price_id` ([`upgrade.php:123`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/upgrade.php)).
2. **Checkout** — the card POSTs to [`checkout.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/checkout.php): `csrf_verify()`, refuse if `COMING_SOON_PAID_PLANS` ([:11](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/checkout.php)) or `!PlatformStripe::enabled()` ([:28](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/checkout.php)), then `PlatformStripe::createCheckoutSession($tenantId, $planId)` ([:34](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/checkout.php)) and redirect to the returned URL.
3. **`createCheckoutSession`** — [`PlatformStripe.php:102-135`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformStripe.php): resolves the plan, refuses if `stripe_price_id` is empty ([:109](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformStripe.php)), `ensureCustomer` (idempotent customer create, tenant_id in metadata, [:65-95](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformStripe.php)), then a `mode=subscription` Checkout Session with `tenant_id`+`plan_id` in both session and `subscription_data.metadata` ([:122-131](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformStripe.php)) — that metadata is how the webhook resolves the sub back to a tenant.
4. **Stripe Checkout** completes → redirects to `return.php?session_id=…` ([:120](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformStripe.php)). [`return.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/return.php) does **not** write the plan — it meta-refreshes every 3s ([:29](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/return.php)) until `currentPlan.slug` is `pro_monthly`/`pro_annual` ([:21](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/return.php)). The flip is the webhook's job.
5. **Webhook flips state** — Stripe POSTs `customer.subscription.created`/`invoice.paid` to `/platform-webhook/stripe/`. `invoice.paid` mirrors `subscriptions.plan_id` onto `tenants.plan_id` ([`index.php:274-277`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/platform-webhook/stripe/index.php)). Once `tenants.plan_id` points at the Pro row, `Plan::canCreate` reads the new (NULL = unlimited) caps and `enforce_plan_cap` stops blocking — Pro "unlocks." There are no feature gates; what changes is brand/URL/capacity ([`upgrade.php:63`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/upgrade.php)).

Cancel/manage: [`manage.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/manage.php) renders current plan + Stripe-portal and cancel forms (only when `$sub && PlatformStripe::enabled()`, [:55](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/manage.php)). [`cancel.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/cancel.php) calls `cancelSubscriptionAtPeriodEnd` ([:24](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/cancel.php)) — Stripe-side only; the local `cancel_at_period_end` flag arrives via the `customer.subscription.updated` webhook.

## The webhook idempotency contract
[`index.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/platform-webhook/stripe/index.php) mirrors the inherited shop-webhook contract (CLAUDE.md invariant 3 / MODELS.md "`subscriptions.local_status` is webhook-driven only", [MODELS.md:628-632](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/design-docs/MODELS.md)):

1. **Refuse if not configured** — `!PlatformStripe::enabled()` returns **503** (not 200) so Stripe retries later rather than silently dropping events ([:24-27](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/platform-webhook/stripe/index.php)).
2. **Verify signature** — `constructWebhookEvent` against `STRIPE_PLATFORM_WEBHOOK_SECRET`; failure → 400 ([:43-49](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/platform-webhook/stripe/index.php)).
3. **Claim the event** — `INSERT INTO public.billing_events (stripe_event_id, …)` **first** ([:61-66](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/platform-webhook/stripe/index.php)). The PK serializes concurrent retries. On duplicate-key (`23505`/`23000`): if `processed_at IS NOT NULL`, ack **200** and stop (real duplicate); else fall through and re-run (a prior attempt crashed) ([:67-84](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/platform-webhook/stripe/index.php)).
4. **Dispatch + stamp in one transaction** — `Database::transaction()` switches on `event->type`, then `UPDATE billing_events SET processed_at = now(), handler_result = ?` ([:88-121](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/platform-webhook/stripe/index.php)). On throw: `processed_at` stays NULL, `handler_result='FAILED'`, return 500 so Stripe retries ([:122-131](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/platform-webhook/stripe/index.php)).

Which event flips what:
| Stripe event | Handler | Effect |
| --- | --- | --- |
| `customer.subscription.created` / `.updated` | `handle_subscription_upsert` ([:144](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/platform-webhook/stripe/index.php)) | INSERT or `transitionTo` the `subscriptions` row; `local_status = mapStripeStatus(sub.status)`; resolves `plan_id` from `items[0].price.id`. |
| `invoice.paid` | `handle_invoice_paid` ([:249](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/platform-webhook/stripe/index.php)) | `transitionTo('ACTIVE')`, roll period_end, **mirror `sub.plan_id` → `tenants.plan_id`**, restore tenant from `GRACE`. |
| `invoice.payment_failed` | `handle_invoice_failed` ([:289](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/platform-webhook/stripe/index.php)) | Only on `attempt_count >= 4` or `next_payment_attempt === null`: `transitionTo('PAST_DUE')` + tenant `ACTIVE → GRACE`. |
| `customer.subscription.deleted` | `handle_subscription_deleted` ([:220](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/platform-webhook/stripe/index.php)) | `transitionTo('CANCELLED')` + flip `tenants.plan_id` back to `Plan::free()`; clear `GRACE → ACTIVE`. |
| anything else | default | marked `IGNORED`, processed so Stripe stops retrying ([:109-112](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/platform-webhook/stripe/index.php)). |

`_map_stripe_status` is [`Subscription::mapStripeStatus`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Subscription.php) ([:37-47](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Subscription.php)): `incomplete→PENDING_PAYMENT`, `active`/`trialing→ACTIVE`, `past_due`/`unpaid→PAST_DUE`, `canceled→CANCELLED`, `incomplete_expired→INCOMPLETE_EXPIRED`, **unknown→CANCELLED (fail closed)** so a Stripe rename can't leave a sub silently ACTIVE.

## Coming-soon gating
`COMING_SOON_PAID_PLANS` is defined from the `PAID_PLANS_COMING_SOON` env var, **default `1`/on** ([`config.php:244-245`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/config/config.php)). Read in three surfaces:
- [`marketing/pricing.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/marketing/pricing.php) — public pricing page renders "Coming soon" pills/copy ([:69,100](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/marketing/pricing.php)).
- [`upgrade.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/upgrade.php) — banner + every paid CTA `disabled` with "Coming soon" label ([:68,123-124,146-147,171-172](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/upgrade.php)).
- [`checkout.php:11`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/billing/checkout.php) — server-side hard stop: even a hand-crafted POST is refused with a flash, so the gate isn't just cosmetic.

Flip to `0` at launch (once live Stripe Price IDs exist); see [LAUNCH_CHECKLIST.md](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/design-docs/LAUNCH_CHECKLIST.md).

## Invariants & gotchas
- **`local_status` / `plan_id` are webhook-driven, never optimistic** — controllers never write either directly. `cancel.php` flips Stripe only; the local mirror waits for `customer.subscription.updated`. Breaking this re-introduces the drift `_map_stripe_status` exists to prevent (CLAUDE.md invariant 3).
- **State changes go through `transitionTo`** — `Subscription::transitionTo` ([:94](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Subscription.php)) validates against `STATUSES`, whitelists writeable columns, and writes a `PlatformActivity` audit row. Don't `UPDATE subscriptions` directly (CLAUDE.md invariant 4).
- **`enabled()` is all-or-nothing** — false until secret + publishable + webhook secret are all set ([:19-24](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformStripe.php)); every method/surface fails shut rather than half-configured.
- **No `stripe_price_id` ⇒ no checkout** — paid plans ship with NULL price; `createCheckoutSession` throws ([:109](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformStripe.php)) and the CTA is disabled until `bin/set-plan-prices.php` wires them.
- **Pinned API version** — `PINNED_API_VERSION = '2023-10-16'` ([:37](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformStripe.php)) **must** match the dashboard endpoint version or payload shapes disagree (STRIPE_AUDIT F3/F6).
- **Idempotency keys** — `ensureCustomer` keys on tenant id ([:88](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformStripe.php)); `refundPayment` takes an optional key ([:186](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformStripe.php)). Checkout/Portal sessions do **not** (STRIPE_AUDIT F4, accepted LOW).
- **dunning threshold** — `handle_invoice_failed` ignores attempts 1–3; GRACE only on the 4th failure or Stripe giving up ([:300-304](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/platform-webhook/stripe/index.php)). The 60-day SUSPENSION sweep is a separate cron, not here.
