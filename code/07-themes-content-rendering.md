# 07 — Themes, content & page rendering

**Audience:** developers working on makerfolio-saas.
**Scope:** the CSS-variable theme system, admin-editable display text, homepage-section gating, the branding footer, and sample-content seeding/reset.

> Public pages are server-rendered with no build step. A small set of static helper classes turn DB/`settings` state into the page: `Theme` remaps `style.css`'s CSS variables from a six-role palette, `PageText`/`EventTypes` resolve editable strings, `PageSections` gates and orders the homepage partials, and `Branding` decides the footer byline. Every customizable value falls back to a code-side default, so a fresh, settings-less install still renders.

## The flow

A public page (e.g. [`public/index.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/index.php)) includes `templates/nav.php`, which calls `Theme::styleBlock()` and `Theme::googleFontsLink()` in `<head>`.

**Theme resolution.** `Theme::current()` ([Theme.php:212](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L212)) reads `theme_preset` (default `terra-gold`, [Theme.php:51](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L51)) from `settings`, looks the preset's six-role palette up in `presets()` ([Theme.php:58](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L58)), then applies any per-role hex overrides from `overridableRoles()` ([Theme.php:34](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L34) — `primary/accent/background/surface/text/cool`). Unknown preset/font/scale keys silently fall back to defaults ([Theme.php:218](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L218), [237](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L237)). `styleBlock()` ([Theme.php:314](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L314)) derives lighter/darker shades via `shade()` and emits a `:root{…}` block that re-binds the *legacy* variable names `style.css` already uses (`--terra`, `--clay`, `--sage`, `--parchment`, …) plus `--font-display`/`--font-body`/`--font-eyebrow` and `--radius`/`--shadow`. One preset thus repaints the whole site without touching the stylesheet.

**Font loading.** `googleFontsLink()` ([Theme.php:371](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L371)) requests only the three active families (+ Caveat for non-eyebrow script accents). The admin page instead uses `previewFontsLink()` ([Theme.php:391](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L391)), which loads *every* themeable family so each dropdown option previews in its own face.

**Live preview.** The editor renders the site in an iframe with `?_theme_preview=<base64-json>`. `previewFromRequest()` ([Theme.php:268](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L268)) base64-decodes the blob, **but only when `Auth::isLoggedIn()`** ([Theme.php:272](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L272)); it re-validates every field against the same allowlists and returns `null` on any decode failure or forged slug, so a malicious blob just falls back to the persisted theme. Nothing is written to the DB during preview.

**Editable text.** `PageText::get('home','featured_work_title')` ([PageText.php:185](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PageText.php#L185)) returns the override row `text.home.featured_work_title` if non-blank, else the `DEFAULTS` entry ([PageText.php:27](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PageText.php#L27)); unknown keys **throw**. Partials wrap every call in `e()` — see [hero.php:81](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/sections/home/hero.php#L81) and [featured-work.php:9](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/sections/home/featured-work.php#L9). Event labels route through `EventTypes::label()` ([EventTypes.php:28](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/EventTypes.php#L28)), which merges the admin `event_type_labels` JSON over `DEFAULT_LABELS` and falls back to `'Event'` for unknown types.

**Section gating.** `index.php` ([index.php:101](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/index.php#L101)) loops `PageSections::enabled('home')` and includes `sections/home/<key-with-dashes>.php` for each visible section in `sort_order`. `enabled()`/`isVisible()` ([PageSections.php:71](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PageSections.php#L71), [84](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PageSections.php#L84)) read the `page_sections` table; a catalog entry with **no DB row defaults to visible** ([PageSections.php:132](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PageSections.php#L132)) and a missing table (pre-migration) falls back to all-defaults ([PageSections.php:121](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PageSections.php#L121)). Inner pages (`about`, `portfolio`, `templates`) use inline `if (PageSections::isVisible(...))` guards. Note partials *also* self-gate on data — `featured-work.php` early-returns when `$featured` is empty ([featured-work.php:4](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/sections/home/featured-work.php#L4)).

**Footer byline.** `templates/footer.php` calls `Branding::footerMode($isSaaS, $tenantId, $allowRemove)` ([Branding.php:41](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Branding.php#L41)): self-host (or apex-with-null-tenant) → `MODE_LICENSE` (the LICENSE-required "Site built by…" byline); SaaS tenant without `allow_remove_branding_footer` → `MODE_POWERED` ("Powered by makerfolio"); paid tenant → `MODE_NONE`.

**Meta.** `PageMeta::renderHead()` ([PageMeta.php:29](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PageMeta.php#L29)) emits description + OG + Twitter tags, resolving image paths to absolute URLs via `SITE_URL`. Pages still write their own `<title>`.

## Adding things (conventions)

- **Theme preset** — add an entry to `presets()` ([Theme.php:58](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L58)) with all six palette roles in `#rrggbb`. `isValidPreset()` and `current()` pick it up automatically; the admin picker reads the same array.
- **Font** — add to `displayFonts()`/`bodyFonts()`/`eyebrowFonts()` ([Theme.php:137](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L137)–[182](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L182)) with `label`, `family` stack, and Google-Fonts `gf` slug. Validators + both font links derive from these arrays.
- **Overridable color role** — add to `overridableRoles()` ([Theme.php:34](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L34)) (role → settings key); `current()`, `previewFromRequest()`, and the admin save handler all loop it, so one edit wires it end-to-end. Don't forget the matching `SETTING_OVERRIDE_*` const and a `--var` consumer in `styleBlock()`.
- **PageText key** — add a key + default under the right group in `DEFAULTS` ([PageText.php:27](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PageText.php#L27)), render it via `PageText::get()`. The admin editor ([`page-text.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/settings/page-text.php)) auto-renders a field grouped by key prefix; optionally list it in `HIGH_IMPACT` ([PageText.php:163](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PageText.php#L163)).
- **Homepage section** — per the header comment ([PageSections.php:19](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PageSections.php#L19)): (1) add a `CATALOG` entry, (2) add `DEFAULT_SORT_ORDER`, (3) seed a `page_sections` row in init.sql + a migration, (4) create the partial (`section_key` snake_case → `section-key.php`). A `PageSectionsTest` enforces (1)/(2)/(4) agree.
- **Event type** — extend the schema ENUM **and** add to both `DEFAULT_LABELS` and `CSS_CLASSES` ([EventTypes.php:13](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/EventTypes.php#L13)).

## Invariants & gotchas
- **Never hardcode display strings.** All public copy goes through `PageText::get()` / `EventTypes::label()`; labels are admin-editable and unknown PageText keys throw — a typo fails loud at render, not silently.
- **CSP nonce on the theme `<style>`.** The strict `style-src` forbids arbitrary inline styles; `styleBlock()` stamps the per-request `csp_nonce()` ([Theme.php:362](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L362)). Without it the browser drops the rule and every tenant's palette silently reverts to default. The admin live-preview `<style>` carries the same nonce ([theme.php:109](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/settings/theme.php#L109)).
- **Preview is admin-gated.** `?_theme_preview` is honored only for logged-in admins and only after re-validating against the allowlists — anonymous or forged blobs are ignored ([Theme.php:272](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L272)).
- **All theme writes go through `validate*()`.** Anything not in an allowlist is rejected before DB or CSS emission ([Theme.php:411](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L411)–[416](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Theme.php#L416)); `styleBlock()` only ever interpolates validated hex/known slugs.
- **SampleContent is purely additive + idempotent.** It never DELETEs; the `sample_content_seeded` marker ([SampleContent.php:31](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SampleContent.php#L31)) short-circuits a second run ([SampleContent.php:108](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SampleContent.php#L108)). SVG placeholder colors are re-validated as 6-digit hex even though they come from code consts, to block `fill="…"` breakout ([SampleContent.php:323](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SampleContent.php#L323)).
- **ContentReset preserves schema/admins/auth/migrations.** DB wipes run in one transaction; filesystem deletes happen *after* commit so a FS failure leaves the DB consistent ([ContentReset.php:219](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ContentReset.php#L219)). The `design` partition UPSERTs defaults (never deletes) and reseeds `page_sections` from `PageSections::DEFAULT_SORT_ORDER` ([ContentReset.php:252](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ContentReset.php#L252)).
- **SaaS vs self-host seed branch.** `SampleContent::seed()` writes `*_storage_key` columns on the SaaS path (tenantId given) and legacy `image_path` on self-host ([SampleContent.php:137](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SampleContent.php#L137)) — Phase 5.1 dropped the legacy columns from tenant schemas.

## Tests & verification
- [`tests/ThemeTest.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/tests/ThemeTest.php) — hex/preset/font/scale allowlists, `shade()` math (clamping, identity, white/black blends), and that **every** preset defines all six roles in valid hex and every font carries a `gf` slug + family. Catches a half-added preset/font.
- [`tests/PageTextTest.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/tests/PageTextTest.php) — `get()` default-vs-override, unknown-key throw, `HIGH_IMPACT` membership.
- [`tests/PageSectionsTest.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/tests/PageSectionsTest.php) — `CATALOG` ↔ `DEFAULT_SORT_ORDER` key parity, every home key has a partial on disk, unknown pair returns false without a DB call, homepage gaps-of-10 convention.
- [`tests/BrandingTest.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/tests/BrandingTest.php) — exhaustive `footerMode()` matrix (2 bools + nullable int) and constant-string pinning.
- [`tests/EventTypesTest.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/tests/EventTypesTest.php) — default labels + `'Event'`/`'event'` fallbacks; [`tests/SampleContentTest.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/tests/SampleContentTest.php) and [`tests/ContentResetTest.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/tests/ContentResetTest.php) cover catalog sanity + table/setting partition lists.
