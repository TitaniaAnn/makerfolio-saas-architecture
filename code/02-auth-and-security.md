# 02 — Auth & security

**Audience:** developers working on makerfolio-saas.
**Scope:** the three auth keyspaces, login (local + OAuth), 2FA, rate limiting, support sessions, roles, and the request-level security contracts (CSRF, hardening headers, CSP).

> One PHP session carries three non-overlapping keyspaces: `platform_admin_id` (the operator team, 2FA-mandatory, `public.platform_admin_users`), `admin_id` (a tenant's own admins, per-tenant schema, 2FA optional), and the anonymous shop customer (no account). Each `requireLogin()` gates on its own key only — being a platform admin grants you nothing on a tenant, and vice versa. The lone bridge is the audited "log in as tenant" support flow, which honors a support session as a tenant login without ever spoofing `admin_id`.

## The flow

**Tenant local login.** `Auth::loginLocal()` ([Auth.php:156](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)) first checks the provider is enabled, then `isRateLimited($ip)` ([Auth.php:309](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)) — a rate-limited IP is refused *without checking the password*, and the attempt is still recorded so attackers stretch their own lockout ([Auth.php:167-173](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)). It accepts the username **or** the account email ([Auth.php:178-185](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)); a missing user/empty hash runs a dummy `password_verify` against a fixed bcrypt string to defeat a user-existence timing oracle ([Auth.php:189](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)). On a correct password it opportunistically rehashes if PHP's default cost moved ([Auth.php:200-204](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)). If `totp_enabled = 1` it does **not** establish the session — it regenerates the session id, stashes `pending_2fa_admin_id` + `pending_2fa_started_at`, and returns `LOGIN_NEEDS_2FA` ([Auth.php:208-217](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)). Otherwise `completeLocalLogin()` finalizes ([Auth.php:232](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)).

**TOTP challenge.** [`public/admin/auth/2fa-challenge.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/auth/2fa-challenge.php) is reached only after `LOGIN_NEEDS_2FA`. `pending2faAdminId()` enforces a 5-minute ceiling on the pending marker ([Auth.php:264-276](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)). Failed codes feed the **same** `login_attempts` IP counter ([2fa-challenge.php:53-57](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/auth/2fa-challenge.php)). The challenge accepts a 6-digit TOTP (`Totp::verifyCode`, ±1 timestep / ~90s window, [Totp.php:63](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Totp.php)) or a 10-char single-use recovery code (`Totp::findRecoveryCode`, [Totp.php:146](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Totp.php)).

**OAuth (GitHub/Google).** `getGitHubAuthUrl()`/`getGoogleAuthUrl()` stash a 16-byte `oauth_state` nonce + provider in the session ([Auth.php:466](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php), [566](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)). The callbacks compare state with `hash_equals` and verify the stashed provider matches before exchanging the code ([Auth.php:496-502](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php), [595-601](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)). GitHub gates on `allowed_users`, Google on a lowercased `allowed_emails` list plus `email_verified` ([Auth.php:519](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php), [620-628](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)). OAuth bypasses local rate limiting (it's throttled upstream by GitHub/Google). Provider availability is settings-backed and per-tenant via `AuthProviders` — the UI hides disabled providers and `Auth` refuses to act on them as defense in depth ([AuthProviders.php:43-50](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/AuthProviders.php)).

**The auth gate.** `Auth::requireLogin()` ([Auth.php:69](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)) redirects to the **host-relative** `/admin/login` so a session timeout bounces to the *current tenant's* login, not the platform apex. On the authenticated path it emits `Cache-Control: private, no-store, …` + `Pragma: no-cache` so admin pages never linger in a shared-device back-button cache or a misconfigured CDN ([Auth.php:88-91](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)). `isLoggedIn()` is `admin_id` **or** an active support session ([Auth.php:59-67](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)).

**Rate limiting + trusted-proxy `clientIp()`.** Five failures per IP in a 600s sliding window locks an IP out ([Auth.php:135-137](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)); the window math is dialect-aware (`make_interval` on pgsql, `INTERVAL` on MySQL, [Auth.php:314-316](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)). A successful login clears the IP's failure rows ([Auth.php:247](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)). All three helpers fail *open* only on a missing `login_attempts` table (SQLSTATE `42S02`) and rethrow any other PDO error ([Auth.php:296-303](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)). `clientIp()` ([Auth.php:395](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)) returns `REMOTE_ADDR` directly **unless** that address is a trusted proxy — only then does it honor `CF-Connecting-IP` then leftmost `X-Forwarded-For`. `isTrustedProxyIp()` trusts loopback plus exact-IP/CIDR entries from `TRUSTED_PROXIES` ([Auth.php:425](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)); without this gate a direct-exposed host could forge XFF to dodge the per-IP limit. The same proxy gate guards `isSecureRequest()`'s reading of `X-Forwarded-Proto` ([Auth.php:48-52](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)).

**Platform admin login.** `PlatformAuth` shares the same PHP session but keys on `platform_admin_id` ([PlatformAuth.php:33-37](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformAuth.php)). 2FA is **mandatory**: `loginLocal()` returns `LOGIN_NEEDS_2FA` on a good password + enrolled TOTP, and `LOGIN_FAILED` if `totp_enabled = 0` — it never returns `LOGIN_OK` ([PlatformAuth.php:76-114](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformAuth.php)). It also requires `is_active = 1`. OAuth callbacks land in the same place: allowlist-check, match an active `platform_admin_users` row by `github_username`/`google_email`, then plant the same `pending_platform_2fa_*` marker and bounce to the TOTP page — OAuth never skips 2FA ([PlatformAuth.php:246](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformAuth.php), [333](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformAuth.php)). `verifyTotpAndComplete()` tries the 6-digit code first, then a recovery code, nulling the consumed slot in place ([PlatformAuth.php:136-178](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformAuth.php)). `requireSuperadmin()` gates the most dangerous actions (log-in-as, migrations) and renders a nonce-styled 403 ([PlatformAuth.php:456](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformAuth.php)).

**"Log in as tenant" via support sessions.** `SupportSession::start()` ([SupportSession.php:27](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SupportSession.php)) requires a non-empty reason, inserts a row storing only `sha256(rawToken)` + a 1-hour expiry, and returns the raw token for a one-time resume URL. The tenant subdomain hits `resumeByToken()` ([SupportSession.php:58](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SupportSession.php)), which validates the token shape, matches on `token_hash` **and** `tenant_id` while non-ended/non-expired, then **nulls `token_hash`** so the URL can't be replayed, and plants `support_session_id` in the tenant's own session. `activeId()` re-checks on every request that the row's `tenant_id` equals `TenantResolver::currentTenantId()` ([SupportSession.php:109-115](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SupportSession.php)) — the cross-tenant cookie-replay defense. `Auth::getUser()` synthesizes a `Support: <name>` display user (id `0`, never a real `admin_users.id`) so the topbar shows the operator, not the impersonated owner ([Auth.php:102-114](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)). `Auth::logout()` ends any in-flight support session explicitly ([Auth.php:125-127](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)).

**Roles.** `Auth::establishSession()` stamps `admin_role` into the session on every login path so `require_role` avoids a per-request DB hit; a lookup failure defaults to `OWNER` so a DB hiccup can't lock an admin out of their own dashboard ([Auth.php:688-694](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)). `require_role()` ([bootstrap.php:411](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/bootstrap.php)) reads the session role, falls back to a one-time DB lookup if the session predates Slice C, and renders a 403 otherwise. `Role`'s predicates are pure and fail closed — a null/unknown role is false on every predicate ([Role.php:19-97](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Role.php)).

## The security contracts

**CSRF.** `csrf_token()` lazily mints a 32-byte token per session ([bootstrap.php:214](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/bootstrap.php)); forms emit it via `csrf_field()`, GET-style deletes pass `?csrf=`. `csrf_verify()` ([bootstrap.php:225](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/bootstrap.php)) checks both `$_POST['csrf']` and `$_GET['csrf']` with `hash_equals`, returns 403 on failure, and only follows `HTTP_REFERER` back when it is **same-origin with the current request host** — comparing scheme + host (port-stripped) against `HTTP_HOST`, *not* `SITE_URL`, because in SaaS mode a tenant subdomain never matches the platform apex. Anything else falls back to the request host's `/admin/dashboard.php` (or `/` for CLI), closing the open-redirect ([bootstrap.php:240-260](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/bootstrap.php)).

**Hardening headers** (every PHP response, [bootstrap.php:113-117](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/bootstrap.php)): `Referrer-Policy: strict-origin-when-cross-origin` (kills the `?csrf=` token-in-referer-log leak on cross-origin nav), `X-Content-Type-Options: nosniff`, `X-Frame-Options: SAMEORIGIN`.

**CSP — no `unsafe-inline`, nonce on `style-src` only.** Phase 7H extracted every inline script and on-handler, so `script-src` is `'self'` + named CDNs with **zero** inline allowance — deliberately *no* nonce, so a future pasted `<script nonce>` can't silently reopen the inline-script door ([bootstrap.php:159-164](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/bootstrap.php)). `style-src` keeps a per-request nonce (`bin2hex(random_bytes(16))`, [bootstrap.php:92-93](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/bootstrap.php)) as the escape hatch for genuinely-dynamic inline `<style>` (Theme per-tenant CSS vars + the four error pages). Also: `object-src 'none'`, `base-uri 'self'`, `form-action 'self' https://checkout.stripe.com`, `frame-ancestors 'self'`, `upgrade-insecure-requests` ([bootstrap.php:165-177](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/bootstrap.php)).

**Uploads PHP-exec lockdown.** Inherited contract: `public/uploads/.htaccess` denies executing `.php*` in the upload tree (see [CLAUDE.md](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/CLAUDE.md) — defense-in-depth against an upload-validation bypass; the SaaS routing tier is Caddy, but the file stays for self-host).

## Invariants & gotchas
- **Three keyspaces, one session, no crossover** — `platform_admin_id` ≠ `admin_id` ≠ shop customer; each `requireLogin()` checks only its own key ([PlatformAuth.php:8-13 docblock](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformAuth.php)). Don't gate a platform page on `admin_id` or vice versa.
- **Platform 2FA is non-optional** — `PlatformAuth::loginLocal` never returns `LOGIN_OK`, and OAuth plants the same pending marker. A `totp_enabled=0` platform row is unloginnable by design ([PlatformAuth.php:100-105](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/PlatformAuth.php)).
- **Support sessions are tenant-pinned** — `activeId()` rejects a valid row whose `tenant_id` ≠ the resolved tenant. A stolen cookie replayed on another subdomain is ignored, *not* cleared (the user may be tab-hopping) ([SupportSession.php:109-115](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SupportSession.php)).
- **`clientIp()` only trusts XFF behind a trusted proxy** — set `TRUSTED_PROXIES` correctly in prod or the rate limit reads the proxy IP; leave it loopback-only on a direct-exposed host or XFF becomes spoofable ([Auth.php:395-435](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)).
- **Rate-limit helpers fail open only for the missing table** — `42S02` returns "not limited"; any other PDO error rethrows so a real fault surfaces rather than silently disabling the limiter ([Auth.php:296-303](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Auth.php)).
- **Role predicates never touch the DB** — they take the role string; the lookup is the caller's job. `require_role` reads `$_SESSION['admin_role']` and a stale/null role fails closed ([Role.php:11-18 docblock](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Role.php), [bootstrap.php:417-433](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/bootstrap.php)).
- **`SignupRateLimit` interpolates its window constant into SQL** — `IP_WINDOW`/`EMAIL_WINDOW` are class constants, never user input, so it's safe; the IP/email values are parameterized ([SignupRateLimit.php:31-54](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/SignupRateLimit.php)). Keep windows as constants.
- **hCaptcha fails open when unconfigured, closed when half-configured** — `verify()` returns true if no sitekey, but returns false (and logs) if a sitekey is set with an empty secret ([HCaptcha.php:34-44](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/HCaptcha.php)).

## Tests & verification
- **`bin/auth-attempts-smoke.php`** — `AuthAttempts` record/ipBlocked/identifierBlocked end-to-end on live Postgres: per-purpose isolation, empty-IP short-circuit, sliding-window expiry.
- **`bin/role-gates-smoke.php`** — provisions a Studio tenant with OWNER/EDITOR/CONTRIBUTOR admins; asserts each `Role::canX` predicate × role, that `establishSession` stamps `admin_role`, and that `require_role`'s DB fallback recovers a pre-Slice-C session.
- **`bin/support-session-smoke.php`** — start writes `sha256(token)` + ~1h expiry; resume is single-use (second resume fails); `activeId` cross-tenant guard; `via_support=1` on writes inside a session; `getUser` shows `Support: <name>` without spoofing `admin_id`.
- **`bin/csp-nonce-smoke.php`** — `csp_nonce()` is a stable 32-char hex string per request, plus a loopback curl confirming the header `nonce-X` matches the `<style nonce="X">` in the same response body.
- **`tests/`** — `AuthClientIpTest`, `AuthSecureRequestTest` (trusted-proxy + XFF/XFP logic), `TotpTest` (RFC 6238 vectors, recovery codes), `RoleTest` (predicate truth table), `SupportSessionTest`, `AuthProvidersTest` / `PlatformAuthProvidersTest` / `PlatformAuthTest`, `HCaptchaTest`, `SignupValidationTest`. These are the pure-helper paths; DB-touching login flows are covered by the smokes above.
