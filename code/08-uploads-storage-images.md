# 08 â€” Uploads, storage & the image pipeline

**Audience:** developers working on makerfolio-saas.
**Scope:** image/file upload, GD resize/crop/rotate/thumbnail, the pluggable `Storage` backend, keyâ†’URL resolution, and the gallery edit/delete handlers.

> Every uploaded byte goes through one `Storage` interface ([`Storage.php`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/Storage.php)) with two implementations â€” `LocalStorage` (disk, dev/self-host) and `S3Storage` (R2/S3, prod) â€” chosen at runtime by `STORAGE_DRIVER` via the `get_storage()` singleton. Rows reference their files only by a `*_storage_key` column shaped `<tenant_id>/<rel>`; `StorageUrl::urlFor()` turns a key into a public URL at render time. Phase 5.1 dropped the legacy `*_path` columns, so the storage key is now the single source of truth on the read path.

## The flow

**Upload (single).** `ImageUpload::upload($file, $subdir)` ([`ImageUpload.php:6`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php)) is the spine:
1. Reject non-`UPLOAD_ERR_OK` and oversize files. The per-file cap is the tenant's plan limit when resolvable (`Plan::tenantUploadBytes`), else `MAX_IMAGE_SIZE` ([`ImageUpload.php:14-23`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php)).
2. `enforceStorageCap()` refuses *before touching disk* if the bytes would push the tenant over `max_storage_bytes` (`Plan::canCreate('storage_bytes', â€¦)`) ([`ImageUpload.php:29`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php), [`:322`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php)).
3. MIME sniff via `finfo` â€” only `image/jpeg|png|webp`; **GIF is intentionally rejected** (no thumbnail branch) ([`ImageUpload.php:34-40`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php)).
4. `uniqid('pottery_', true)`-derived filename, `move_uploaded_file` into `UPLOAD_PATH/<subdir>` ([`ImageUpload.php:42-61`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php)).
5. `resizeOriginalIfLarger()` shrinks the in-place original to `MAX_ORIGINAL_DIMENSION` on the longer edge ([`ImageUpload.php:67`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php), [`:170`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php)); `createThumbnail()` writes `thumb_<filename>` at `THUMB_WIDTHĂ—THUMB_HEIGHT` ([`ImageUpload.php:72-74`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php), [`:122`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php)).
6. **Dual-write to Storage** when a tenant is resolved: build keys with `StorageUrl::keyForTenant`, `put()` both original + thumb. A storage failure is logged and **nulls the keys** so the DB never claims a key absent from the bucket ([`ImageUpload.php:80-110`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php)).
Returns `path/thumb/url/thumb_url/storage_key/thumb_storage_key` ([`ImageUpload.php:112-119`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php)); the controller persists `storage_key`/`thumb_storage_key`.

**Multi-file.** `MultiFileUpload::parse($_FILES['k'])` reshapes the parallel arrays into per-file arrays *keyed by original index* (so a caller can correlate POSTed labels), skipping empty/errored slots ([`MultiFileUpload.php:18`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/MultiFileUpload.php)). Each entry then feeds `ImageUpload::upload`.

**Display.** Templates read only `*_storage_key` and call `StorageUrl::urlFor($key)` â†’ public URL, or `''` for a null key (templates already treat empty `src` as "no image") ([`StorageUrl.php:75`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/StorageUrl.php)).

**Crop / rotate / delete (gallery edits).** The three handlers share an identical guard shape: whitelist `imagesTable`/`parentTable`/`parentIdColumn` against fixed literals (PG can't parameterise identifiers), load the row by `id + parentIdColumn`, mutate, repoint the gallery row, and *if `is_primary`* repoint the parent's cached cover keys ([`ImageCropHandler.php:42`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageCropHandler.php), [`ImageRotateHandler.php:39`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageRotateHandler.php), [`ImageDeleteHandler.php:51`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageDeleteHandler.php)).
- **Crop** maps fractional rect (0..1, clamped) onto the object's real pixels, crops via `cropImageFile`, *regenerates* the thumb from the cropped original (aspect ratio changed), writes both under **new** keys, repoints, then drops the old objects best-effort ([`ImageCropHandler.php:117-165`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageCropHandler.php)).
- **Rotate** turns original + thumb a quarter turn (`rotateImageFile`), writes under new keys, repoints, drops old ([`ImageRotateHandler.php:65-105`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageRotateHandler.php)).
- **Delete** drops both storage objects best-effort, deletes the row, then promotes the next image (`sort_order, id`) as primary and syncs the parent; `blockLastImage` refuses deleting the only image ([`ImageDeleteHandler.php:69-149`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageDeleteHandler.php)).

`hasThumbSibling` distinguishes `piece` (parent carries `image_thumb_storage_key`) from `products` (parent has `image_storage_key` only) â€” set per endpoint ([`pieces/crop-image.php:16`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/pieces/crop-image.php) vs [`shop/crop-image.php:17`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/admin/shop/crop-image.php)). Endpoints are thin: `requireLogin` â†’ `require_role(OWNER, EDITOR)` â†’ (shop) `enforce_plan_flag('allow_shop', â€¦)` â†’ `csrf_verify()` â†’ handler â†’ `json_encode`.

**Downloadable files.** `TemplateFileUploader::upload($file)` ([`TemplateFileUploader.php:21`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/TemplateFileUploader.php)) validates extension+MIME against its own allowlist, enforces the storage cap, moves into `public/uploads/templates/files/`, dual-writes to Storage, and returns `file_path/file_name/file_size/file_ext/file_storage_key`.

## Storage backends

`get_storage()` ([`bootstrap.php:288`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/bootstrap.php)) is a static singleton: `STORAGE_DRIVER=s3` â†’ `S3Storage`, anything else â†’ `LocalStorage`.

- **LocalStorage** maps keys 1:1 to paths under `UPLOAD_PATH` (default root via `defaultRoot()`). `put` does an **atomic** copy-to-`.tmp.<rand>`-then-rename to avoid a partial-file race ([`LocalStorage.php:41-50`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/LocalStorage.php)); `get` realpath-confirms the file stays under root before copying to a temp ([`LocalStorage.php:62-64`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/LocalStorage.php)); `delete` is idempotent ([`LocalStorage.php:81`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/LocalStorage.php)); `listPrefix` recursively walks and normalises separators to `/` ([`LocalStorage.php:106`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/LocalStorage.php)).
- **S3Storage** uses `aws/aws-sdk-php` (lazily constructed) with `use_path_style_endpoint` (R2/MinIO) and `ACL: public-read` on `put` ([`S3Storage.php:52-74`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/S3Storage.php)). `listPrefix` paginates `ListObjectsV2` with the continuation token ([`S3Storage.php:123-138`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/S3Storage.php)). `STORAGE_TEST_FAKE=1` transparently delegates every method to an internal LocalStorage rooted at a unique temp dir, so smokes/tests exercise the full contract with no network ([`S3Storage.php:31-40`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/S3Storage.php)).
- **URL resolution.** `StorageUrl::publicUrl()` prefers `MEDIA_URL_BASE` (prod R2 behind a CDN hostname), else falls back to `UPLOAD_URL` (Caddy serving the mounted `/uploads/` volume) ([`StorageUrl.php:56-66`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/StorageUrl.php)).
- **Backfill.** `bin/migrate-uploads-to-storage.php` is the Phase-5 one-shot that pushes pre-Phase-5 on-disk files into Storage and stamps `*_storage_key`; idempotent, per-tenant try/finally with `Database::resetSchema` between tenants ([`migrate-uploads-to-storage.php:1-20`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/bin/migrate-uploads-to-storage.php)).

## Invariants & gotchas

- **Keys start with `<tenant_id>/`** â€” `keyForTenant()` enforces it and is the cross-tenant isolation boundary (listing a prefix counts only that tenant's bytes). It rejects empty/absolute paths, `\0`, backslashes, and any `.`/`..`/empty segment ([`StorageUrl.php:34-53`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/StorageUrl.php)). `tenantUsageBytes` relies on this prefix shape.
- **Edits write NEW keys, never overwrite.** `StorageUrl::urlFor` has no cache-busting, so crop/rotate mint a fresh random basename and repoint the row; the new URL forces the browser/CDN to refetch ([`ImageCropHandler.php:8-9`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageCropHandler.php), [`ImageRotateHandler.php:6-8`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageRotateHandler.php)). Old objects are dropped best-effort â€” **a failed delete must not undo a successful edit** (orphan keys are cheap to GC).
- **Storage writes are best-effort on the write path, authoritative on read.** Upload/edit dual-writes log+null on failure rather than aborting the on-disk move ([`ImageUpload.php:96-108`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php)). But post-5.1 there is **no disk read-fallback**: a null `image_storage_key` renders broken until the backfill repopulates it.
- **Identifier whitelists, not parameters.** Table/column names are interpolated into SQL (Postgres can't bind identifiers), so all three handlers hard-check against `SAFE_*` literal lists; today's callers pass hardcoded strings, the check is defensive against a future caller routing user input ([`ImageDeleteHandler.php:14-24`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageDeleteHandler.php)).
- **GIF unsupported by design** â€” no `IMAGETYPE_GIF` branch in `createThumbnail`; uploads of `image/gif` are rejected at MIME check ([`ImageUpload.php:31-40`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php)).
- **`ImageUpload::delete()` traversal guard** â€” refuses `..`, `\0`, and any realpath escaping `UPLOAD_PATH`; also removes the sibling `thumb_` file ([`ImageUpload.php:345-371`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php)).
- **`.htaccess` PHP-exec lockdown** â€” `public/uploads/.htaccess` denies `.php/.phtml/.phar/.pht/.php[3-8]` and disables the PHP handler/type, defeating `image.jpg.php` double-extension tricks ([`.htaccess:12-27`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/public/uploads/.htaccess)). Self-host/Apache only; the SaaS Caddy config achieves the same separately.
- **`TemplateFileUploader` limits**: `ALLOWED_EXTS = pdf,png,jpg,jpeg,webp`, `ALLOWED_MIMES = application/pdf,image/png,image/jpeg,image/webp`, `MAX_SIZE = 10485760` (10 MB). **SVG (script-bearing) and ZIP are intentionally excluded** ([`TemplateFileUploader.php:9-15`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/TemplateFileUploader.php)).
- **Storage cap fails open.** Any exception inside `Plan::canCreate` is logged and the upload proceeds â€” a billing glitch must not block real work ([`ImageUpload.php:329-334`](https://github.com/TitaniaAnn/makerfolio-saas/blob/main/includes/ImageUpload.php)).
